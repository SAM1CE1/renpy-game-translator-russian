Real-time translation

The Base version of NovaTextVN picks up game text during gameplay and gradually builds useful cache without a separate mandatory prep stage.

How it works in practice

01

The game shows a new line

During a scene, the player sees an original line or interface text that has not been handled yet.

02

NovaTextVN intercepts the line

The text goes through the active translation path and is prepared for in-game replacement.

03

The translated result is shown

Once the line is processed, the player keeps reading the scene in Russian without leaving the game.

04

The line is stored in cache

If the same text appears again later, NovaTextVN can reuse the known result more quickly.

What the player actually sees

After installation, the game still launches normally. Translation is enabled from the F10 menu and then works directly during reading.

  • The NovaTextVN menu opens with F10.
  • Translation is enabled without a separate launcher.
  • Pre-scan remains an optional helper, not a strict requirement.
  • Repeated lines get faster thanks to cache.

Why cache matters here

After the first pass, known lines can open faster because NovaTextVN already has stored results for them.

Why this is useful

No need to wait for full preparation

You can start playing right after installation and initial setup.

Good for the first playthrough

The Base mode is useful when you mainly want to understand the story and characters quickly.

Pairs well with pre-scan

If the game is long, you can warm up the cache and make later reading smoother.

The player stays inside the game

Translation is controlled through the familiar NovaTextVN menu instead of an external workflow.
